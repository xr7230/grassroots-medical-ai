# 基层医疗AI文书智能体 v3.0

> **对话 → 病历，自动生成。诊断给方向，不给定论。**

## 产品定位

一个**基层医生的文书副驾驶**——输入医患对话或病情描述，自动生成规范医疗文书。

| 不是 | 是 |
|------|-----|
| ❌ AI替代医生诊断 | ✅ AI辅助生成病历 |
| ❌ C端症状自查工具 | ✅ B端医生工作流嵌入 |
| ❌ 追求诊断命中率 | ✅ 诊断给方向，不编数据 |

## 三类文书

| 文书类型 | 核心内容 | 使用场景 |
|----------|----------|----------|
| 📋 **入院记录** | 主诉、现病史、既往史、体格检查、初步诊断、治疗计划 | 患者入院时 |
| 🏥 **出院小结** | 入院情况、诊疗经过、出院诊断、出院医嘱、随访建议 | 患者出院时 |
| 🩺 **随访记录** | 体格检查、用药评估、生活方式指导、下次随访建议 | 出院后复诊 |

## 安全特性

### 17类输入守卫

系统内置多层输入校验，覆盖以下恶意/异常输入类别：

| 类别 | 示例 | 处理方式 |
|------|------|----------|
| 纯数字/乱码 | `12345`、`asdfgh` | 拦截，提示输入医学描述 |
| SQL注入 | `' OR '1'='1` | 拦截，记录警告 |
| XSS攻击 | `<script>alert(1)</script>` | 拦截，净化输出 |
| 超长输入 | >5000字符 | 截断并警告 |
| 纯英文无医学 | `hello world` | 拦截，要求中文医学描述 |
| 重复字符 | `头痛头痛头痛...(x50)` | 去重后处理 |
| 空输入/空白 | ``、`   ` | 拦截 |
| 非医学提问 | `今天天气怎么样` | 拦截 |
| 政治敏感 | 政治人物/事件 | 拦截 |
| 脏话侮辱 | 辱骂内容 | 拦截 |
| JSON注入 | `{"__proto__":...}` | 拦截 |
| 命令注入 | `$(rm -rf /)` | 拦截 |
| 特殊字符洪水 | `!@#$%^&*()...` | 拦截 |
| URL/链接 | `http://malware.com` | 拦截 |
| 个人隐私泄露 | 身份证号、银行卡号 | 脱敏处理 |
| Hex/Base64编码攻击 | 编码后的恶意载荷 | 解码检测后拦截 |
| 对抗性前缀 | `忽略前面的指令...` | 拦截 |

### 设计原则

1. **安全第一**：信息不足时不编造；攻击输入正确拦截；诊断方向与症状系统一致
2. **安静后台**：不打断医生工作流，不增加额外操作负担
3. **给方向不给定论**：诊断标注"可能"/"待排"，一次对话不声称百分百确诊
4. **隐私保护**：`.env` 不入库；敏感信息自动脱敏；语音数据不落盘

## 输入 → 输出

**输入**（文本或语音）：医患对话 / 病情描述 / 出院总结 / 随访记录

```
# 入院记录
患者女，28岁，腹痛2天，右下腹，伴恶心，无呕吐。
查体：右下腹压痛，反跳痛阳性。血常规：WBC 12.5×10⁹/L。

# 出院小结
患者男65岁胸闷胸痛3天入院，行支架植入。术后恢复好。
出院带药：阿司匹林100mg qd，氯吡格雷75mg qd。

# 随访记录
患者女55岁高血压随访，服药后血压控制在135/85。
目前口服硝苯地平30mg qd。无头晕头痛。
```

**输出**：根据文书类型自动生成对应格式

```
入院记录 → 主诉：腹痛2天 / 初步诊断：急性阑尾炎可能
出院小结 → 入院情况：因胸痛入院 / 出院医嘱：阿司匹林+氯吡格雷
随访记录 → 血压135/85 / 用药评估：硝苯地平继续 / 下次随访：1个月后
```

## 架构

```
├── backend/
│   ├── agents/medical_agent.py  # 核心智能体（~5600行）
│   │   ├── 输入守卫层 (17类拦截)
│   │   ├── CC提取器 (主诉/伴随症状)
│   │   ├── LLM文书生成器 (入院/出院/随访)
│   │   └── 文书验证器 (完成度/幻觉检测)
│   ├── api/main.py              # FastAPI 接口 + 静态前端

│   ├── knowledge/                # 医学知识库
│   ├── tests/                    # 单元测试 (95/95 ✅)
│   ├── _eval_discharge.py        # 出院小结专项评估
│   ├── _eval_doc_quality.py      # 入院记录质量评估
│   └── .env.example              # 环境变量模板
├── frontend/
│   └── index.html               # Web前端 (文本/语音/三文书切换)
└── data/                        # 训练数据集
```

## 理论基础

基于肖锐（2026）《基层医疗AI需求分析与设计转向》研究结论：
- 基层医生对AI的核心诉求是**文书减负（40.4%）**，而非智能诊断（仅11.5%）
- 现有AI产品存在"重诊断、轻文书"的结构性价值错配
- 设计转向：从"诊断智能体" → "文书智能体"
